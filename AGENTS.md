# 项目协作规则

1. 之前已经完成且正确的功能，尽量不要修改。当前任务如果是完善功能 A，就专注功能 A，不顺手改功能 B。
2. 生成的注释使用中文，并保持 UTF-8 编码。
3. 修改后需要检查新增中文是否有乱码；如果发现乱码，需要修正。
4. 如果修改某个函数实现，先理解原有逻辑，再在原有基础上调整，尽量保留已有正确逻辑。
5. 当前项目运行环境是 Windows。
6. 用户没有明确要求时，不新增专门测试脚本，也不新增专门项目说明文档。
7. 写代码时不考虑 fallback。
8. 代码中不要使用 emoji。
9. 输出使用中文。
10. 在 `D:\AI\fangwu` 项目下进行网页或登录相关测试时，必须使用 Google Chrome 的 `韬` 配置文件。使用 Chrome 自动化时，优先选择 `metadata.profileName === "韬"` 的 Chrome Extension 后端，并接管已打开的 `http://127.0.0.1:5173/` 标签页；不要使用默认 Chrome 后端、隐藏浏览器或新建其他 Chrome 配置，除非用户明确同意。
